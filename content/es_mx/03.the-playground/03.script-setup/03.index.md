# Script setup

Let's break down the example code provided here.

First of all, we are looking at the **Composition API**. The :tooltip-trigger{id='setup-hook'}[`setup()`] method is the telltale.

So far you've been introduced to one way of writing the app:

```file:/src/App.vue title=App.vue /setup()/ {"The returned values will be available in the template":9-12} {"Note the double curly braces to print the value":18} {"Using plain CSS":24-26}
-
```

Another, **more commonly found in the wild** way of writing this is to leverage the `<script setup>` [sytactic sugar](https://vuejs.org/api/composition-api-setup.html#composition-api-setup) that the Vue compiler gives us. So the entire app can be written as:

::magic-move{lang="vue" }

```vue title="With script setup"
<script setup>
import { ref, version } from 'vue'

const message = ref('Hola playground script setup!')
console.log('version:', version)
</script>

<template>
  <h1>{{ message }}</h1>

  Version is: <code>{{ version }}</code>
</template>

<style>
h1 {
  color: #42b983;
}
</style>
```

```vue title="Without script setup"
<script>
import { ref, version } from 'vue'

export default {
  setup() {
    const message = ref('Hola playground!')
    console.log('version:', version)

    return {
      message,
      version,
    }
  },
}
</script>

<template>
  <h1>{{ message }}</h1>

  Version is: <code>{{ version }}</code>
</template>

<style>
h1 {
  color: #42b983;
}
</style>
```

I like a lot the "less is more" feel to it.

Since we have it available in `playground` given that it _compiles the code_ we write, that is the way we will be writing our code from now on.

::tooltip-content{id="setup-hook"}

Particularly useful when:

- Using Composition API without a build step;
- Integrating with Composition-API-based code in an Options API component.

There is more info about this on the [official docs](https://vuejs.org/api/composition-api-setup.html#composition-api-setup)
